# 2. Npm 패키지 추가하기

{% hint style="info" %}
프로젝트 폴더 최상단 위치에서 아래 명령을 실행해주세요.&#x20;

```
yarn add adwhale-sdk-react-native
  또는
npm install adwhale-sdk-react-native
```



아래 링크에서도 추가 방법을 확인할 수 있습니다.

[https://www.npmjs.com/package/adwhale-sdk-react-native](https://www.npmjs.com/package/adwhale-sdk-react-native)
{% endhint %}

명령어를 실행하고 나면 package.json에서 다음과 같이 패키지가 추가된 것을 확인할 수 있습니다.

```json
  "dependencies": {
    "adwhale-sdk-react-native": "^2.7.300"
  },
```

### 최신 ReactNative version : 2.7.300

### Version Policy

본 SDK는 Android 네이티브 SDK 버전과의 대응 관계를 명확히 유지하기 위해 다음과 같은 버전 규칙을 사용합니다.

* 앞 세 자릿수(Major.Minor.Patch) 는 Android SDK 버전과 동일
* React Native 전용 수정 사항은 Patch 값을 확장하여 관리
* iOS SDK 대응버전은 [7.-changelog.md](7.-changelog.md "mention") 을 참고하여 주시기 바랍니다.

### Version Usage Guide

프로젝트 상황에 따라 아래 방식을 권장합니다.

#### Android SDK 버전에 고정하여 사용하는 경우 (권장)

운영 안정성을 중시하는 경우

```
{
  "adwhale-sdk-react-native": "~2.7.300"
}
```

* Android SDK 2.7.3 기반 유지
* RN 핫픽스만 자동 반영
* `>=2.7.3+0 <2.8.0` 범위

#### 특정 버전을 완전히 고정하여 사용하는 경우

```
{
  "adwhale-sdk-react-native": "2.7.300"
}
```

* 버전 변경 없음
* 재현성, 장애 분석에 최적
* QA / 장애 재현 환경
* 장기 유지 브랜치

#### Troubleshooting

Android 빌드 오류 발생 시:

```
cd android
./gradlew clean
```

#### 환경점검

```
npx react-native doctor
```
